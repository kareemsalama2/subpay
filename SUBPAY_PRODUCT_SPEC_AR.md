# SubPay - مواصفات المنتج الاولية

## الفكرة

SubPay هو تطبيق لادارة رومات الاشتراكات المشتركة. كل روم تمثل اشتراك او خدمة معينة، مثل ChatGPT او Netflix او خدمة داخل شركة. الادمن يضيف المستخدمين، يتحكم في وصولهم، ويتابع الدفع الشهري. الروم تعرض بيانات الاشتراك والاشعارات المهمة مثل OTP او رسائل الخدمة، مع امكانية ارسال رسائل وملفات للاعضاء.

## الشاشات الموجودة في التصميم

- Splash: شاشة بداية باسم SubPay.
- تسجيل الدخول: بريد الكتروني، كلمة مرور، دخول بجوجل، رابط انشاء حساب، ورابط نسيان كلمة السر.
- انشاء حساب جديد: اسم المستخدم، رقم الهاتف، البريد الالكتروني، كلمة المرور، دخول بجوجل.
- تغيير كلمة السر: ارسال كود للبريد، ثم ادخال الكود وتعيين كلمة مرور جديدة.
- الصفحة الرئيسية: قائمة الرومات المتاحة للمستخدم وزر دخول روم جديدة.
- دخول روم جديدة: ادخال كود مرسل عبر البريد الالكتروني.
- صفحة الروم: تبويبات لبيانات الروم/الاشعارات والمستخدمين.
- حالة عدم الدفع: منع الوصول للروم مع زر دفع.
- صفحة الدفع: اختيار طريقة دفع مثل InstaPay او محافظ/بطاقات.
- صفحة المستخدمين: قائمة اعضاء الروم مع حذف المستخدمين.
- الملف الشخصي: بيانات المستخدم، الدفع، الاشعارات، معلومات عنا، تسجيل الخروج، حذف الحساب.
- تعديل الملف الشخصي: تعديل الاسم، البريد، الصورة، وكلمة السر.
- FAQ / من نحن: موجودة لكن محتواها حاليا من مشروع اخر باسم Dexter وتحتاج اعادة كتابة.

## Flow المستخدم

1. المستخدم يسجل حساب او يدخل بحسابه.
2. يدخل على الصفحة الرئيسية ويرى الرومات التي هو عضو فيها.
3. لو يريد دخول روم جديدة، يضغط "دخول روم جديدة".
4. النظام يطلب كود دعوة/تحقق ارسله الادمن او ارسله النظام للبريد.
5. بعد التحقق، تظهر الروم في الصفحة الرئيسية.
6. عند فتح الروم:
   - لو دفع الاشتراك الشهري، يرى بيانات الروم والاشعارات والاعضاء.
   - لو لم يدفع، تظهر شاشة منع الوصول وزر الدفع.
7. المستخدم يدفع، ثم يتم تفعيل الوصول للشهر الحالي.

## Flow الادمن

1. ينشئ روم جديدة باسم الخدمة.
2. يضيف بيانات الاشتراك: البريد، اسم الخدمة، الملاحظات، وربما كلمة المرور.
3. يضيف المستخدمين بالايميل او يرسل لهم كود دعوة.
4. يحدد سعر اشتراك كل عضو وحالة الدفع الشهرية.
5. يرسل رسائل او ملفات للروم.
6. يراقب من دفع ومن لم يدفع.
7. يحذف مستخدم او يوقف وصوله.

## MVP المقترح

- Authentication: تسجيل/دخول/نسيان كلمة المرور.
- Rooms: عرض الرومات، دخول روم بكود، تفاصيل الروم.
- Members: اضافة وحذف اعضاء بواسطة الادمن.
- Payments: حالة دفع شهرية لكل عضو داخل كل روم.
- Room access gate: منع مشاهدة الروم اذا لم يتم الدفع.
- Room notices: اشعارات وبيانات الاشتراك داخل الروم.
- Profile: عرض وتعديل بيانات المستخدم.
- FAQ/About: محتوى صحيح عن SubPay.

## Features لاحقة

- Mailbox integration لاستقبال OTP والرسائل داخل الروم.
- File uploads داخل كل روم.
- Chat داخلي بين الادمن والاعضاء او رسائل ادمن فقط.
- Push notifications.
- Admin dashboard كامل لادارة كل الرومات.
- صلاحيات متعددة: Owner, Admin, Member.
- Logs لمعرفة من شاهد OTP او كشف كلمة السر.

## ملاحظات UI مهمة

- توحيد اللغة: حاليا يوجد خلط بين العربي والانجليزي مثل "Your rooms" و"Join new room". يفضل نسخة عربية كاملة او انجليزية كاملة.
- اتجاه النص: الشاشات عربية، فالاتجاه الاساسي يجب ان يكون RTL، مع معالجة الايميلات وكلمات المرور كـ LTR داخل الحقول.
- الصفحة الرئيسية تحتاج عنوان اوضح: "روماتي" بدل "Your rooms".
- زر "دخول روم جديدة" ممكن يبقى "انضمام لروم" او "ادخال كود روم".
- شاشة FAQ يجب تغيير محتواها بالكامل، لانها تتكلم عن Dexter وحجوزات.
- شاشة الروم تحتاج تبويب واضح باسم "الاشعارات" او "بيانات الاشتراك"، وليس الاثنين مختلطين.
- شاشة المستخدمين تحتاج زر اضافة عضو واضح للادمن.
- قائمة المستخدمين داخل الروم وصفحة "المستخدمين" متشابهتين؛ يفضل تحديد هل واحدة للعرض وواحدة للادارة.

## ملاحظات امنية

- لا يفضل عرض كلمة المرور صريحة لكل الاعضاء افتراضيا.
- يجب استخدام زر اظهار/اخفاء لكلمة السر، مع تسجيل الحدث في logs.
- OTP يجب ان يكون له صلاحية زمنية قصيرة داخل الروم.
- يجب تحديد من يستطيع رؤية بيانات الاشتراك: كل الاعضاء ام الادمن فقط.
- حذف الحساب يجب ان يحتوي تأكيد واضح.
- حذف عضو من روم يجب ان يمنع وصوله فورا للاشعارات والملفات.

## محتوى FAQ بديل

### ما هو SubPay؟

SubPay هو تطبيق يساعد المجموعات والشركات على ادارة الاشتراكات المشتركة داخل رومات منظمة، مع متابعة الدفع والاعضاء والاشعارات المهمة.

### كيف ادخل روم؟

الادمن يرسل لك كود دعوة على البريد الالكتروني. بعد ادخال الكود، تظهر الروم في حسابك.

### لماذا لا استطيع فتح الروم؟

اذا لم يتم دفع الاشتراك الشهري، يتم ايقاف الوصول مؤقتا حتى اتمام الدفع.

### هل يستطيع كل الاعضاء رؤية بيانات الاشتراك؟

هذا يعتمد على اعدادات الادمن. يمكن جعل بعض البيانات ظاهرة للجميع وبعضها للادمن فقط.

### هل يمكن استخدام SubPay داخل شركة؟

نعم، يمكن استخدامه كرومات داخلية لارسال رسائل وملفات وتنظيم الوصول لمجموعات معينة.
